

## Fix: PDF Brand Colors Not Rendering

### Root Cause
The color constants in `src/lib/pdf-brand.ts` are declared `as const` (readonly tuples). When spread with `...PDF_PRIMARY` into jsPDF methods like `setFillColor(r, g, b)`, TypeScript rejects the spread because jsPDF expects mutable number arguments, not a readonly tuple. This causes a compile error, so the old unbranded code is served from cache.

### Fix

**`src/lib/pdf-brand.ts`** — Change color constant types from `as const` readonly tuples to plain number arrays:

```typescript
// Before (breaks spread into jsPDF methods)
export const PDF_PRIMARY = [101, 143, 64] as const;

// After (works with spread)
export const PDF_PRIMARY: [number, number, number] = [101, 143, 64];
export const PDF_ACCENT: [number, number, number] = [217, 175, 47];
export const PDF_DARK: [number, number, number] = [30, 41, 21];
export const PDF_LIGHT_BG: [number, number, number] = [243, 240, 233];
```

This single change fixes all four files since they all import and spread these same constants.
